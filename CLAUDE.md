# CLAUDE.md — ForgeDesk

## Project Overview

ForgeDesk is a self-hosted service desk and project management platform for software consulting. Clients manage projects (briefs, features, bugs, Gantt charts) through a simple UI. An internal orchestrator dispatches work to AI coding agents (Claude, Gemini, Codex, Mistral) running on a K8s cluster.

**Roles:** superadmin (global), staff (global, agent controls), client (per-org, no agent internals visible).

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Language | Go 1.25 (`go.mod`), built with `golang:1.24-alpine` + `GOTOOLCHAIN=auto` |
| Router | `go-chi/chi/v5` |
| Database | PostgreSQL via `jackc/pgx/v5` (no ORM, raw SQL) |
| Migrations | `golang-migrate/migrate`, files in `migrations/` (26 migrations) |
| Auth | Argon2id passwords, TOTP 2FA (`pquerna/otp`), server-side sessions, RBAC |
| Templates | Go `html/template`, SSR |
| Interactivity | HTMX (`hx-boost="true"` on body, `historyCacheSize: 0`), Alpine.js |
| CSS | Hand-written CSS in `static/css/app.css` — **no Tailwind** |
| JS | Vanilla JS, bundled via `scripts/bundle.sh` into `static/js/bundle.js` |
| Gantt chart | Vanilla JS (no D3) — `static/js/app/gantt.js` |
| AI assistant | Gemini API (`google.golang.org/genai`), SSE streaming, 15 function-calling tools |
| Markdown | `yuin/goldmark` + `alecthomas/chroma` syntax highlighting, EasyMDE editor |
| File storage | AWS S3 via `aws-sdk-go-v2` |
| WebSocket | `gorilla/websocket` for real-time chat |

## Development Workflow

```bash
source .secrets                            # Load env vars (DB, API keys, AES key)
go run ./cmd/server                        # Web app on :8080
go run ./cmd/orchestrator                  # Agent dispatcher (separate process)
sh scripts/bundle.sh                       # Rebuild JS bundle after editing static/js/
migrate -database $DATABASE_URL -path migrations up  # Run migrations
```

**No Tailwind build step.** Edit `static/css/app.css` directly.

## Build & Deploy

```bash
# Build
docker build -t registry.k3s.vlah.sh/smartpm:latest -f deploy/Containerfile .

# Push
docker push registry.k3s.vlah.sh/smartpm:latest

# Deploy
kubectl apply -f deploy/k8s/
kubectl rollout restart deployment/forgedesk-server -n smartpm
```

The Containerfile runs `scripts/bundle.sh` during build, so JS bundling happens automatically.

## Testing

### Unit & Integration Tests

Require a running PostgreSQL instance:

```bash
docker compose -f docker-compose.test.yml up -d postgres
docker compose -f docker-compose.test.yml up migrate
go test ./internal/... -p 1 -count=1 -timeout 120s
```

- **`-p 1` is required** — packages share a test database, parallel runs cause data conflicts
- `-count=1` disables test caching
- `-timeout 120s` — Argon2id hashing is slow in tests

### E2E Tests (Playwright)

```bash
docker compose -f docker-compose.test.yml up -d --build   # Full stack on port 8081
cd e2e && npx playwright test                               # Run all
npx playwright test --debug                                 # Step-through debugger
```

Test suites: `01-auth`, `02-dashboard-orgs`, `03-projects`, `04-tickets`, `05-admin`

**HTMX testing gotchas:**
- After form submissions: `await page.waitForLoadState('networkidle')` — `hx-boost` makes forms AJAX
- Alpine.js dropdowns: use `.evaluate()` to bypass `x-show` transitions
- `testutil.SetupTestDB(t)` handles cleanup in FK-safe order

## Data Model

```
Organization → OrgMembership (owner|admin|member) → User
Organization → Project → Ticket → Comment
                                → TicketActivity (audit log)
                                → Attachment (S3)

Ticket: type (epic|task|subtask|bug), status (backlog→ready→planning→...→done|cancelled)
        priority (low|medium|high|critical), date_start/date_end (Gantt), parent_id (hierarchy)
        agent_mode (null|plan|implement), agent_name (null|claude|gemini|codex|mistral)

User: role (superadmin|staff|client), Argon2id password, TOTP 2FA (mandatory)
      totp_secret (AES-encrypted), recovery_codes (Argon2id-hashed, 10 single-use)
```

---

## Auth & 2FA

- **2FA is mandatory.** No authenticated route accessible without verified 2FA
- Login flow: password → session created → redirect to `/verify-2fa` (or `/setup-2fa` on first login)
- `/setup-2fa` is a dead-end page (no nav, no sidebar) — only way out is completing setup
- Session has two phases: `authenticated` (password OK) → `2fa_verified` (full access)
- TOTP window: ±1 time step (30s tolerance). Recovery codes: 10, single-use, Argon2id-hashed
- WebAuthn: UI stubbed ("coming soon"), code skeleton exists
- Session: 256-bit random token, stored server-side in DB, 24h sliding expiry
- Cookie flags: `HttpOnly`, `Secure` (based on APP_URL scheme), `SameSite=Strict`

## HTMX Patterns

- `<body hx-boost="true">` intercepts all links/forms as AJAX
- History cache disabled (`historyCacheSize: 0`) to prevent stale back-navigation
- Status changes: `hx-patch` → handler returns `HX-Refresh: true`
- Comment posting: `hx-post` → `hx-target="#comments"` + `hx-swap="beforeend"`
- Org switcher: `hx-boost="false"` to allow real form POST

## Coding Conventions

- **Errors:** return, don't panic. Wrap with `fmt.Errorf("context: %w", err)`
- **DB queries:** raw SQL in `internal/models/queries.go` via pgx. No ORM
- **Naming:** `snake_case` DB columns, `CamelCase` Go. `db:"column_name"` struct tags
- **Templates:** `{{define "content"}}` inherits from `{{template "base" .}}`
- **Partials:** standalone template files, no layout wrapping
- **JS:** vanilla only. No transpilation. Alpine.js inline via `x-data`
- **CSS:** hand-written utility classes + component styles. No preprocessor
- **Migrations:** `000001_description.up.sql` / `.down.sql`. Use `ON DELETE CASCADE` for children
- **UUIDs:** all tables use UUID primary keys (`google/uuid`)

### Git Conventions

- Agent branches: `agent/{ticket-id}-{short-slug}`
- Human branches: `feature/`, `fix/`, `chore/`

## Security Rules

- All routes behind `AuthMiddleware` except: `/login`, `/setup-2fa`, `/verify-2fa`, `/static/*`
- Client data isolation: enforce `WHERE org_id = ?` at query level, never trust client context
- TOTP secrets encrypted at rest (AES-256-GCM, key from `AES_ENCRYPTION_KEY` env var)
- `.secrets`, `*.crt`, `*.key` must be in `.gitignore` — never commit
- On password/2FA change: invalidate all sessions for that user
- Agent API keys in `.secrets` only, injected as K8s Secrets

## Gotchas

- **JS bundling order matters:** `scripts/bundle.sh` concatenates with semicolons. Alpine must be last (it auto-initializes on load)
- **HTMX `HX-Refresh: true`** causes full `location.reload()` — breaks HTMX history. Used for status/edit handlers as a fallback
- **`go.mod` says Go 1.25 but Containerfile uses `golang:1.24-alpine`** — this works because `GOTOOLCHAIN=auto` downloads the needed toolchain
- **Test DB shared across packages** — always use `-p 1` flag to prevent parallel test execution
- **`toJSON` template func** returns `template.JS` (not `template.HTML`) for safe script embedding
- **First registered user** is auto-promoted to superadmin
- **Org context** stored in session (`selected_org_id`), switched via POST `/switch-org`
